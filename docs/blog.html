July 22, 2026

Translation moved to cloud inference — a beta tradeoff, explained honestly

Quick follow-up to yesterday's note about moving translation onto our own hardware: we tried it, and our own server's real-world response time for a single translation came in around 50 seconds. That's not usable for an app that's supposed to feel like a conversation.

So translation now runs on real, fast cloud inference instead — three independent translators checking each other plus a referee model, same approach as before, just hosted on hardware built for speed rather than our own server. Responses are now well under two seconds.

The honest tradeoff: this costs real money per translation instead of being free to us. This app is in beta, and if usage ever grows enough to make that cost a real problem, that's genuinely a good problem to have — it means people are using it. We'll say so plainly here if that ever happens, rather than quietly degrading quality or adding something sketchy to cover it.

July 21, 2026

v1.2.0 Windows download fixed, translation now runs on our own hardware

The Windows download button was serving a broken file — it was missing pieces it needed to actually run, so it would fail to open right after downloading. That's fixed now; the download works.

Also, translation for both the Windows app and the Android app now runs entirely on our own server instead of a third-party cloud AI service. Same three-model cross-checking approach as before (two independent translators plus a referee model that catches disagreements between them), just running on hardware we control instead of paying a cloud provider per request.
